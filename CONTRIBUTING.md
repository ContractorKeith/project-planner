# Contributing

Thanks for your interest in improving project-planner!

## How to contribute

1. **Open an issue first** — describe the problem or idea before writing a PR.
   This saves everyone time and lets us align on approach.
2. **Fork and branch** — work on a feature branch, not main.
3. **Keep changes focused** — one issue per PR. Small PRs get reviewed faster.

## What makes a good contribution

- **New skills** — planning capabilities that aren't covered yet.
  Use `skills/skill-creator.md` as a guide for structure.
- **Improved prompts** — if you find the orchestrator or a skill gives weak results,
  suggest specific wording changes.
- **Bug fixes** — typos, broken references, unclear instructions.
- **Example outputs** — real planning sessions (anonymized) that show the tool in action.

## What to avoid

- Don't add code or build tooling — this is a planning-only repo.
- Don't restructure the skill format without discussing it first.
- Don't add dependencies — there are none and we'd like to keep it that way.

## Style

- Keep language direct and imperative in skill files.
- One question at a time — that's a core design principle, even in docs.
- If you're unsure, look at the existing skills for tone and structure.
